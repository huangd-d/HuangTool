# 页签管理功能修改 - 产品需求文档

## Overview
- **Summary**: 修改页签管理功能，包括首页页签不可关闭、初始只打开首页页签、以及从首页打开其他页面时创建新页签的功能。
- **Purpose**: 优化用户体验，确保首页作为基础页签始终可用，同时提供便捷的页面切换方式。
- **Target Users**: 使用该 Web 应用的所有用户。

## Goals
- 首页页签不可关闭，去除页签上的关闭图标
- 应用启动时只打开首页页签，其他页签不自动打开
- 从首页点击接口调试、文档预览、文件预览时，分别打开新页签显示对应页面

## Non-Goals (Out of Scope)
- 不修改页签的样式风格
- 不修改其他页面的内容和功能
- 不修改应用的整体结构

## Background & Context
- 现有应用启动时会自动打开多个页签，包括首页、接口调试、文档预览和文件预览
- 所有页签都可以关闭，包括首页
- 从首页点击其他功能时，直接切换到对应页签，而不是创建新页签

## Functional Requirements
- **FR-1**: 首页页签不可关闭，去除关闭图标
- **FR-2**: 应用启动时只打开首页页签
- **FR-3**: 从首页点击接口调试时，打开新页签显示接口调试页面
- **FR-4**: 从首页点击文档预览时，打开新页签显示文档预览页面
- **FR-5**: 从首页点击文件预览时，打开新页签显示文件预览页面

## Non-Functional Requirements
- **NFR-1**: 页签管理功能的修改不影响其他功能的正常运行
- **NFR-2**: 页签操作的响应速度快，用户体验流畅
- **NFR-3**: 代码结构清晰，易于维护

## Constraints
- **Technical**: 基于 Vue 3 框架，使用 Composition API 和 `<script setup>` 语法
- **Business**: 保持现有功能不变，只修改页签管理逻辑
- **Dependencies**: 无额外依赖，使用现有技术栈

## Assumptions
- 首页页面已经存在，并且包含指向其他页面的链接
- 其他页面（接口调试、文档预览、文件预览）已经存在
- 用户接受新的页签管理逻辑

## Acceptance Criteria

### AC-1: 首页页签不可关闭
- **Given**: 打开 Web 应用
- **When**: 查看首页页签
- **Then**: 首页页签上没有关闭图标，且无法关闭
- **Verification**: `human-judgment`

### AC-2: 应用启动时只打开首页页签
- **Given**: 启动 Web 应用
- **When**: 查看页签栏
- **Then**: 只显示首页页签，其他页签未打开
- **Verification**: `human-judgment`

### AC-3: 从首页打开新页签
- **Given**: 打开 Web 应用，进入首页
- **When**: 点击接口调试、文档预览或文件预览链接
- **Then**: 分别打开新页签显示对应页面
- **Verification**: `human-judgment`
